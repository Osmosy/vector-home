# Hermes Skills для Home Assistant

Адаптировано из Codex Skills CCOSTAN/Home-AssistantConfig.

## 1. homeassistant-dashboard-designer

Задача: спроектировать и сгенерировать Lovelace-дашборд для Home Assistant.

Инструкция агенту:
- Спроси у пользователя: какие сущности, какой тип дашборда (обзорный / по комнатам / по функциям)
- Изучи существующий `ui-lovelace.yaml` если доступен
- Предложи структуру: вкладки, карточки, layout
- Сгенерируй YAML с корректными entity_id из existing конфигурации
- Предупреди о несуществующих entity_id (если пользователь указал несуществующие)
- Выдай готовый YAML + инструкцию по вставке в `ui-lovelace.yaml`

## 2. homeassistant-yaml-dry-verifier

Задача: валидировать HA-конфигурацию без деплоя.

Инструкция агенту:
- Прочитай все `.yaml` файлы в `config/`
- Проверь: синтаксис YAML, ссылки на entity_id, соответствие schema
- Найди: несуществующие entity_id, дублирующиеся unique_id, битые шаблоны Jinja
- Выдай отчёт: ошибки (стоп-деплой), предупреждения (проверить), рекомендации
- Никогда не модифицируй файлы без явного разрешения

## 3. infrastructure-doc-sync

Задача: синхронизировать документацию инфраструктуры с реальным состоянием HA.

Инструкция агенту:
- Извлеки из конфигурации: список интеграций, устройства, сервисы, зависимости
- Сравни с существующей документацией (`docs/` или `README.md`)
- Найди расхождения: интеграции в конфиге, но не в доке; устаревшие описания
- Сгенерируй обновлённую документацию в Markdown
- Включи: версию HA, количество сущностей, топ автоматизаций, критические зависимости

## 4. network-architecture-diagrammer

Задача: построить диаграмму сети умного дома на основе HA-конфигурации.

Инструкция агенту:
- Извлеки из конфигурации все IP-адреса, хосты, интеграции с сетевыми устройствами
- Определи: VLAN, подсети, протоколы (MQTT, Zigbee, Z-Wave, Wi-Fi, Bluetooth)
- Построй Mermaid-диаграмму: устройства → протокол → хаб → интеграции → облака
- Выдели: критические пути (без них ничего не работает), точки отказа
- Добавь легенду и рекомендации по сегментации

## 5. homeassistant-automation-debugger (новый)

Задача: проанализировать логи автоматизаций и найти причины сбоев.

Инструкция агенту:
- Прочитай `home-assistant.log` или логи конкретных автоматизаций
- Найди: автоматизации, которые не сработали (trigger был, action — нет)
- Для каждой проблемной автоматизации: покажи trigger, condition, action → где разрыв
- Предложи исправление: либо YAML-патч, либо объяснение почему condition не прошёл
- Выдели: автоматизации, которые срабатывают слишком часто (потенциальный флуд)

## Установка в Hermes

Скопировать в `~/.hermes/skills/smart-home/` и использовать:
```
/skill homeassistant-dashboard-designer
/skill homeassistant-yaml-dry-verifier
/skill infrastructure-doc-sync
/skill network-architecture-diagrammer
/skill homeassistant-automation-debugger
```

Или через `hermes -s homeassistant-automation-debugger "проанализируй логи за сегодня"`.
